# Cybernet AI · Merch Vault
## Стратегический продуктовый документ

> Внутренний геймифицированный мерч-маркет для команды Cybernet AI.
> Сотрудники зарабатывают **CyberCoins (CC)** за вклад в компанию и обменивают их на брендированный мерч.

---

## 0 · TL;DR

**Что строим:** не интернет-магазин, а замкнутую экосистему мотивации. Покупка мерча — это последний шаг длинного цикла "вклад → CC → статус → дроп". Дизайн и UX должны подчёркивать **эксклюзивность** и **членство в клубе**, а не "ещё один корп-портал".

**Тон:** AI-нативный, минималистичный, технологичный. Тёмная тема как дефолт. Лайм (`#D7FF1A`) — единственный сильный акцент. Никаких stock-иллюстраций, фиолетовых градиентов и Lorem Ipsum.

**Стек (рекомендация):** Next.js 15 (App Router) + React Server Components + Tailwind + shadcn/ui на фронте, NestJS / FastAPI на бэке, Postgres + Redis, S3 для медиа, Auth.js или Clerk для SSO.

**Roadmap (90 дней):** MVP за 4 недели (auth + каталог + корзина + чекаут), геймификация + лидерборд за 8 недель, интеграция с источниками CC за 12.

### 1.2 Иерархия и навигация

**Глобальная нав** (sticky topbar): Дашборд · Магазин · Рейтинг · Профиль. Корзина и баланс — справа, всегда на виду. На мобильном — нав в гамбургере.

**Принцип:** баланс CC должен быть виден **на каждом экране**. Это якорь экосистемы — пользователь всегда понимает, на что он может рассчитывать.

## 2 · UX-флоу

### 2.1 First-time user (онбординг нового сотрудника)

```
HR создаёт аккаунт → email-инвайт
    ↓
[Email] "Welcome to Cybernet Vault. Your starter pack: 500 CC"
    ↓
Клик → /login?invite=xxx
    ↓
Sign up: имя/команда уже предзаполнены, нужен только пароль
    ↓
Confirmation overlay: "Стартовый бонус +500 CC начислен"
    ↓
/app (Dashboard) — приветственный hero, hint-tooltip "это твой баланс"
    ↓
[Push 24ч] "Загляни в магазин — 500 CC хватит на стикерпак"
```

**Цель онбординга:** показать ценность валюты в первые 5 минут. Стартовый бонус → возможность что-то купить сразу → ощущение "система работает".

### 2.2 Returning user · core loop

```
Логин (SSO в один клик)
    ↓
Dashboard
    │
    ├─→ Видит "+80 CC за code review #2401" в активити
    ├─→ Видит "ты #7, поднялся на 3 позиции"  (дофамин)
    └─→ Видит "Cyan Edition Hoodie · NEW"   (FOMO)
    ↓
[Click на товар]
    ↓
Product modal → выбирает размер/цвет → "В корзину"
    ↓
Toast confirmation + cart-button pulse
    ↓
Продолжает шопиться ИЛИ открывает корзину
    ↓
[Cart drawer]
    │
    ├─ Если хватает CC → активная кнопка "Обменять"
    └─ Если не хватает → красный warning "Не хватает 320 CC"
                            + suggestion "за этот спринт можно заработать ещё 200 CC"
    ↓
[Checkout]
    ↓
Success overlay: списание, остаток, "Зашибись"
    ↓
[Push] "Заказ принят. Получишь когда мерч приедет в офис"
    ↓
[Email через 5-7 дней] "Твой мерч готов. Зайди к Севаре в OPS"
```

### 2.3 Edge cases (что обработать)

| Сценарий | Решение |
|---|---|
| Не хватает CC в корзине | Inline-warning + дизейбл checkout + подсказка "как заработать" |
| Дроп распродан | Бейдж "SOLD OUT" + кнопка "Notify on restock" |
| Возврат / отмена заказа | Auto-refund CC + запись в истории |
| Сотрудник увольняется | Графс-период 30 дней на использование баланса, потом списание |
| Случайный double-click чекаута | Идемпотентный чекаут — заказ создаётся ровно один раз |
| Балланс ушёл в минус (баг) | Hard-constraint в БД, alert в Sentry |

### 2.4 Геймификация (что мотивирует возвращаться)

Это основной отличающий фактор от обычного e-com. Три параллельных слоя мотивации:

1. **Транзакционная мотивация** — "хочу худи, надо ещё 320 CC"
2. **Соревновательная** — "хочу подняться в рейтинге Q2"
3. **Коллекционная** — "хочу ачивку Drop Hunter (купить 5 limited)"

**Tier-система** (пример):
- Tier 1 · `RECRUIT` (0–2 000 CC заработано) — базовый каталог
- Tier 2 · `OPERATOR` (2 000–5 000) — +5% скидка на дропы
- Tier 3 · `ARCHITECT` (5 000–15 000) — ранний доступ к новым дропам за 24ч до общего
- Tier 4 · `OPERATIVE` (15 000–35 000) — эксклюзивные товары (cap edition)
- Tier 5 · `LEGEND` (35 000+) — кастомизация (своё имя на бейдже худи)

**Ачивки** (триггерные награды, дают CC + статус):
- 🎯 First Drop · первая покупка · +50 CC
- 🔥 Streak Keeper · 7 дней активности · +100 CC
- 🏆 Hackmaster · победа в хакатоне · +800 CC
- 🐋 Whale · 20 000 CC в кошельке одновременно · эксклюзивный бейдж

**Sources of CC** (как зарабатывают — настраивается админом):
- Code review · 20 CC за PR (auto через GitHub webhook)
- Shipped feature · 250 CC (manual из Linear)
- Helping juniors · 15 CC за отмеченную сессию (peer rec)
- Hackathon win · 800/400/200 CC (1st/2nd/3rd, manual)
- Tech talk · 150 CC (за внутренний доклад)
- Bug bounty · 100–500 CC (по severity)
- Welcome bonus · 500 CC (one-time)
- Anniversary · 1000 CC × лет (yearly)

### 3.4 Product page (модалка vs страница)

**Гипотеза:** модалка > страница. Причины:
- Не теряется контекст шопинга (можно быстро открывать-закрывать товары).
- Меньше нагрузка на роутинг и кэш.
- Лучше ощущается "лёгкость" покупки.

Но **должна быть и страница**: для прямых ссылок (кто-то делится в Slack), для SEO внутреннего поиска, для индексации в админке. Решение: модалка по умолчанию, `/app/shop/:slug` как fallback при прямом заходе.

## 4 · Дизайн-система

### 4.1 Палитра

#### Тёмная тема (default)

| Токен | HEX | Где используется |
|---|---|---|
| `--bg-0` | `#0A0A0B` | основной фон страницы |
| `--bg-1` | `#111114` | карточки, поверхности |
| `--bg-2` | `#18181C` | вложенные блоки, инпуты |
| `--bg-3` | `#22222A` | hover, элементы UI |
| `--line` | `#2A2A33` | разделители, бордеры |
| `--line-strong` | `#3D3D48` | акцентные бордеры |
| `--text` | `#F5F5F5` | основной текст |
| `--text-muted` | `#9A9AA8` | вторичный текст |
| `--text-dim` | `#6A6A78` | служебные подписи |
| `--lime` | `#D7FF1A` | **главный акцент** — баланс, CTA, лого |
| `--cyan` | `#5EEAD4` | вторичный акцент — градиенты, тиры |
| `--coral` | `#FF4D6D` | удаление, ошибки, корзина-бейдж |
| `--gold` | `#FACC15` | 1-е место в рейтинге, ачивки |
| `--success` | `#34D399` | начисления CC, успешные действия |
| `--warning` | `#FBBF24` | предупреждения |

#### Светлая тема (рекомендация для phase 2)

Если решите делать светлую — **не инвертировать тёмную**. Сделать отдельную с тёплой бумажной базой:

| Токен | HEX |
|---|---|
| `--bg-0` | `#F8F7F2` (off-white, чуть тёплый) |
| `--bg-1` | `#FFFFFF` |
| `--text` | `#1E1B4B` (логотип navy) |
| `--lime` | `#A8C913` (приглушённый, чтобы не слепил) |
| `--accent-deep` | `#1E1B4B` (как в логотипе) |

**Почему:** обычная инверсия даёт скучный "корпоративный белый". Тёплый off-white + navy-deep — это редактирский стиль, который выглядит дорого.

### 4.2 Типографика

| Семейство | Использование | Веса |
|---|---|---|
| **Unbounded** (display) | заголовки, баланс, бренд | 700, 800, 900 |
| **Sora** (body) | основной текст, UI | 300, 400, 500, 600, 700 |
| **JetBrains Mono** (mono) | числа, таги, "// 0x01", технические подписи | 400, 500, 700 |

**Шкала размеров (px):**

| Роль | Размер | Используется |
|---|---|---|
| Hero | 56–96 | "Earn. Flex. Repeat.", баланс |
| H1 | 32–48 | Заголовки страниц |
| H2 | 22–28 | Заголовки секций |
| H3 | 16–18 | Заголовки карточек |
| Body | 14–15 | Основной текст |
| Small | 13 | Подписи в списках |
| Caption | 11–12 | Лейблы форм, теги, таймстемпы |
| Micro | 10 | Бейджи, статусы |

**Tabular numbers** (`font-variant-numeric: tabular-nums`) обязательны для всех чисел в UI: баланс, CC-цены, рейтинг, qty в корзине. Без этого числа "прыгают" при изменении.

### 4.3 Spacing и радиусы

**Spacing scale (8px base):** 4 · 8 · 12 · 16 · 20 · 24 · 32 · 40 · 56 · 80 · 120

**Радиусы:**
- 6px — мелкие чипы и иконки
- 12px — кнопки, инпуты
- 20px — карточки
- 28px — большие hero-блоки

**Никаких радиусов меньше 6px** — выглядит дёшево. Никаких больше 32px (кроме круглых) — выглядит игрушечно.

### 4.5 Иконография

**Не использовать** Heroicons / Material — слишком generic. Минимально-кастомизированный набор от **Lucide** (что использовано в прототипе) либо вообще свой 16-иконочный набор в одной стилистике (1.5px stroke, rounded-linecap).

### 4.6 Дизайн-принципы (что отличает Cybernet от Shopify-копипасты)

1. **Один акцентный цвет.** Лайм используется только для:
   - Баланса CC
   - Главного CTA
   - Логотипа
   - Активного состояния навигации
   Если ставите лайм везде — он перестаёт работать.

2. **Числа главнее картинок.** Это про "8 920 CC", "+12% мес/мес", "#7 из 142". Числа — главный язык приложения.

3. **Mono-теги вместо capslock.** `// CATALOG · LIVE` вместо `CATALOG (LIVE)`. Это даёт ощущение терминала / системы / клуба разработчиков.

4. **Густая, но дышащая сетка.** Background-grid из 56×56 квадратов с radial-mask — придаёт "чертёжное" ощущение без шума.

5. **Микро-движение в "статичных" элементах.** Пульсирующая зелёная точка, вращающийся пунктир вокруг монеты, ambient-glow в hero. Без этого UI "мёртвый".

## 5 · Анимации и микровзаимодействия

### 5.1 Easings (использовать только эти)

```css
--ease:      cubic-bezier(0.22, 1, 0.36, 1);      /* default */
--ease-snap: cubic-bezier(0.6, 0, 0.2, 1);        /* для быстрых "щелчков" */
```

Никаких `ease-in-out` или `linear` для UI.

### 5.2 Длительности

- Hover/focus state — **150ms**
- Modal/drawer open — **300–400ms**
- Page transitions — **400–500ms**
- Page-load stagger — **50ms между элементами**, общая длина 600ms

### 5.3 Конкретные взаимодействия

| Триггер | Что происходит |
|---|---|
| **Page load** (Dashboard) | Hero-карточка fade-up 400ms, потом stat-cards stagger по 50ms, потом продукты, потом ниже |
| **Hover на product card** | Lift `translateY(-4px)` + усиление shadow + smooth border |
| **Click "В корзину"** | Toast slide-in (right→left, 300ms) + cart-button **pulse** scale 1→1.18→1 |
| **Открытие drawer** | Backdrop fade 300ms + panel slide-in 400ms |
| **Открытие modal** | Backdrop fade + panel scale 0.96→1 + opacity 0→1 |
| **Cart count change** | Badge pop (scale 0→1.2→1, 200ms) |
| **Hover на кнопку primary** | Lift 1px + усиление glow |
| **Click на quick-add (+)** | Кнопка крутится `rotate(90deg)` |
| **Checkout success** | Лайм-кружок с галочкой `scale(0)→1.15→1` + radial glow ambient |
| **Tier progress bar** | Загрузка `0% → 62%` за 800ms на первом рендере |
| **Achievement unlock** | Конфетти 1.5s + sound effect (опц.) + toast |

### 5.4 Чего избегать

- **Спиннеры.** Заменять skeleton-loaders.
- **Параллакс на скролле.** Это 2018-й.
- **Длинные анимации длиннее 600ms** — UI начинает ощущаться медленным.
- **Анимация на каждом элементе.** Только на тех, где это рассказывает историю.

## 6 · Технический стек и архитектура

### 6.1 Frontend

**Рекомендация:** Next.js 15 (App Router) + React 19 + TypeScript.

**Почему именно это:**
- **Server Components** — каталог рендерится на сервере → быстро, SEO-дружелюбно (для админки).
- **Server Actions** — мутации без отдельного API-слоя на простых операциях (добавить в корзину, лайк).
- **Edge runtime** — auth-чеки и публичные endpoint'ы.
- **Огромная экосистема** — shadcn/ui, Vercel hosting, Auth.js.

**Ключевые библиотеки:**

| Задача | Что использовать |
|---|---|
| Стили | Tailwind CSS + CSS-переменные для токенов (как в прототипе) |
| Компоненты | shadcn/ui (форк, не зависимость) |
| Анимации | Motion (бывший Framer Motion) + CSS для простых случаев |
| Иконки | Lucide |
| Формы | React Hook Form + Zod-валидация |
| Стейт сервера | TanStack Query (для всего, что приходит с API) |
| Стейт клиента | Zustand для корзины и UI-флагов (не Redux) |
| Таблицы | TanStack Table (для админки + лидерборда) |
| Чарты (доп.) | Recharts |

**Не использовать:** styled-components, Material-UI, Ant Design, Chakra. Все они тянут стилевой багаж и противоречат custom-дизайну.

### 6.2 Backend

**Рекомендация:** один из двух путей.

**Путь A · NestJS (Node)** — если команда фронтендеров и хочет один язык:
- TypeScript end-to-end
- Decorators, DI — структурированно для среднего проекта
- Prisma ORM → typesafe-запросы

**Путь B · FastAPI (Python)** — если есть AI-команда, которая будет интегрироваться:
- Быстрая разработка
- Легко добавить ML (рекомендации товаров, анти-фрод по транзакциям)
- SQLAlchemy + Alembic для миграций

**Структура API:**

```
POST   /auth/login           SSO callback / email-password
POST   /auth/refresh         

GET    /me                   профиль + баланс + тир
GET    /me/transactions      ?cursor=...&type=in|out
GET    /me/achievements
PATCH  /me/wishlist          { productId, action: 'add'|'remove' }

GET    /products             ?category=&priceMin=&priceMax=&sort=
GET    /products/:slug       детали + варианты + остатки

POST   /cart/items           { productId, variantId, qty }
PATCH  /cart/items/:id       { qty }
DELETE /cart/items/:id
GET    /cart                 текущая корзина

POST   /orders               чекаут (idempotency-key обязателен!)
GET    /orders               история
GET    /orders/:id

GET    /leaderboard          ?period=quarter|all-time
GET    /leaderboard/me       контекст: ±5 человек вокруг тебя

# Admin
POST   /admin/transactions   ручное начисление CC
GET    /admin/orders         ?status=pending
PATCH  /admin/orders/:id     { status: 'fulfilled'|'cancelled' }
POST   /admin/products       CRUD
```

### 6.3 База данных

**Postgres** (Supabase / Neon / RDS — что удобнее).

Ключевые таблицы:

```sql
users (id, email, name, team, role, tier, hire_date, ...)
wallets (user_id PK, balance INT, lifetime_earned INT, lifetime_spent INT)
transactions (id, user_id, type, amount, source, ref_id, idempotency_key UNIQUE, created_at)
products (id, slug UNIQUE, name, category, base_price, status, created_at)
product_variants (id, product_id, size, color, sku UNIQUE, stock INT)
cart_items (id, user_id, variant_id, qty, UNIQUE(user_id, variant_id))
orders (id, user_id, total_cc, status, idempotency_key UNIQUE, created_at)
order_items (order_id, variant_id, qty, price_cc)
achievements (id, slug, name, description, icon, criteria JSONB)
user_achievements (user_id, achievement_id, unlocked_at, PRIMARY KEY(user_id, achievement_id))
```

**Критически важно:**
- `idempotency_key` на checkout — иначе double-click создаст два заказа.
- Транзакция баланса — **атомарная**: списание + создание заказа в одной DB-транзакции.
- `balance` хранить как integer (целые CC, без копеек).
- Constraint `CHECK (balance >= 0)` — невозможно уйти в минус.

## 8 · Риски и mitigations

| Риск | Mitigation |
|---|---|
| **Инфляция CC** — все накопили слишком много, мерч улетает за 1 день | Ежемесячный аудит ratio earn/spend; ввод "expiring CC" (CC сгорают через год); dynamic-pricing на дропы |
| **Манипуляции с начислениями** — кто-то закрывает фейковые PR | Все авто-начисления > 100 CC требуют ревью; rate-limit (max 500 CC/день из автоматических) |
| **Низкое использование** — все игнорируют систему | Стартовый бонус + onboarding-tooltip; еженедельный digest "ты заработал 250 CC за неделю"; Slack-напоминания |
| **Логистика мерча** — заказали, а на складе нет | Реальные остатки в БД; pre-order с уведомлением "придёт через 2 нед" |
| **Утечка интерналов** | Закрыть админку и API за корп-VPN; SSO-only; rate-limits; аудит-логи всех админ-действий |
| **Burn-out от соревновательности** | Опциональный анонимный режим в лидерборде; убрать leaderboard для команд, где это вредит культуре |

## 9 · KPI / метрики успеха

**Активация (90 дней):**
- 80%+ сотрудников залогинились хотя бы раз
- 50%+ сделали хотя бы одну покупку
- DAU/MAU ≥ 0.3 после первой недели

**Engagement:**
- Среднее время сессии 2–4 минуты (это не Netflix, но и не туда-сюда)
- Возвратность: ≥ 60% возвращаются в течение 7 дней после первой покупки

**Бизнес:**
- Расход компании на мерч в год ≤ X (управляемый бюджет)
- NPS внутреннего инструмента ≥ 50
- Сокращение текучки (сравнить cohort с до-внедрения и после) — главная мета-метрика, ради которой это вообще делается

## 10 · Что НЕ делать (антипаттерны)

❌ **Не превращать в реальный e-commerce.** Никаких "промокодов", "распродаж", "связанных товаров через recommended-engine". Это **внутренний клуб**, не магазин.

❌ **Не давать менять CC на деньги или что-то вне системы.** Как только CC становится почти-валютой — начинаются юридические вопросы, налоги, серые рынки внутри компании.

❌ **Не геймифицировать всё.** Не давать ачивки за "залогинился 100 раз" — это унижает. Награды только за **реальный вклад**.

❌ **Не делать публичный лидерборд токсичным.** Опциональный анонимный режим обязателен. Не показывать "минусы" в дельте больше чем за неделю — никто не помнит, что упал.

❌ **Не пушить уведомления чаще 1 раза в неделю.** Это внутренняя система, а не TikTok.

❌ **Не делать тёмную тему "просто инверсией" светлой.** Если делаете обе — это разные дизайны с разной палитрой и разным mood.
